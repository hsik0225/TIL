MonsterRace 를 설계할 때 클래스를

GameCneter, GameMachine, MonsterRace, User, Input, Output 으로 나누어보려고 했다

순서는

1. GameCenter에 GameMachine, User 생성
2. GameMachine에 있는 MonsterRace가 생성된다.
3. MonsterRace가 문구를 띄우면 User가 Input을 한다
4. MonsterRace 실행

으로 하려 했는데 User와 GameMachine은 독립적이어야 하는데 자꾸 독립적이지 않게 되었다

예를 들어, User가 Input을 하는 건 <즐거운 몬스터 레이스> 라는 문구가 출력되고 나서인데

그 순간을 User가 Output을 갖지 않는 이상 알 수가 없었다

또 User가 Output을 갖게 되면 User가 출력하게 되는 것이니 User의 행동과 맞지 않았다

그러므로 User Class를 없앨 수 밖에 없었다

더 좋은 설계를 만들 수 있었는데 약간 아쉬움이 남는다
